В ходе выполнения Кейс-задачи №3 были выявлены риски некорректного ввода данных (например, отрицательная зарплата или будущий год приема).
Способы решения:

Внедрение валидации в сеттеры (setSalary, setHireYear) для проверки бизнес-логики.

Использование паттерна «Фабрика» для создания объектов сотрудников.

Замена консольного интерфейса на графический (Qt) или интеграция с API «Пульса».
